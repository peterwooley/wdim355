Implicit and Explicit Binding

  • An excellent resource is A List Apart's Getting Out of Binding Situations in JavaScript
  • In many languages, the 'this' keyword is either:
    • not necessary, or
    • will always refer to the object where it was called.
  • When you can trust the 'this' keyword, you can consider that implicit binding, it's assumed.
  • In JavaScript, there is no implicit binding.
  • The 'this' keyword in JavaScript is both required and tricky.

More Implicit and Explicit Binding

  • To refer to variables only in the object's local scope, you must prepend the 'this' reference.
  • In order to maintain scope when your 'this' value gets out-of-hand, you can use a "lexical closure": 
    function MyObject() {
	var self = this;
	this.name = "A beter object!";
	
	this.calledByEvent = function(arg) {				
		// When called by an event, 'this' refers to another object
		alert("This keyword: " . this.name);
		
		// So, you can reference the private 'self' variable
		alert("Self keyword: " . self.name);
		
		// Alert out Argument, for good measure
		alert("Argument: " . arg);
	}
}

var myobject = new MyObject;
setTimeout(myobject.calledByEvent, 500);

apply & call

  • You can utilize two built-in methods that will allow you to set the scope that should be used for the 'this' keyword
  • apply 
    myobject.calledByEvent.apply(window, ['Window Scope']);
myobject.calledByEvent.apply(myobject, ['myobject Scope']);
  • call 
    myobject.calledByEvent.call(window, 'Window Scope');
myobject.calledByEvent.call(myobject, 'myobject Scope');
