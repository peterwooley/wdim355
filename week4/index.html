Events

  • jQuery has extensive support for events beyond what we've played with, including the ability to bind multiple events, unbind events, and trigger custom events.
  • We can utilize the bind() and trigger() methods for these advanced uses.
  • You can use the optional data parameter of the bind method to pass along data into the event.data scope.
  • All jQuery objects can call any custom event.

JavaScript Best Practices

  • Google has released their JavaScript Style Guide and it's full of good advice!
  • We'll take a look at some of them, but you should become familiar with most of them.

Always Use Semicolons

MyClass.prototype.myMethod = function() {
  return 42;
}  // No semicolon here.

(function() {
  // Some initialization code wrapped in a function to create a scope for locals.
})();

var x = {
  'i': 1,
  'j': 2
}  // No semicolon here.

[normalVersion, ffVersion][isIE]();

var THINGS_TO_EAT = [apples, oysters, sprayOnCheese]  // No semicolon here.

-1 == resultOfOperation() || die();

Do Not Use Multiline String Literals

The whitespace at the beginning of each line can't be safely stripped at compile time; whitespace after the slash will result in tricky errors; and while most script engines support this, it is not part of ECMAScript.

var myString = 'A rather long string, an error message \
actually that just keeps going and going -- an error \
message to make the Energizer bunny blush (right through \
those Schwarzenegger shades)! Where was I? Oh yes, \
you\'ve got an error and all the extraneous whitespace is \
just gravy.  Have a nice day.';

Don't use IE Conditional Compilation

They are often unnecessary and can break tools that are used to minify your JavaScript.

var f = function () {
	/*@cc_on if (@_jscript) { return 2* @*/  3; /*@ } @*/
};

JavaScript Style Rules

  • In general, style names in this fashion:
    • functionNamesLikeThis
    • variableNamesLikeThis
    • ClassNamesLikeThis (e.g., Objects)
    • methodNamesLikeThis
    • SYMBOLIC_CONSTANTS_LIKE_THIS
  • Prefer ' over ".
  • Most importantly: be consistent.
