Chainability (The Magic of jQuery)

  • "In a nutshell: Every method within jQuery returns the [jQuery] object itself, allowing you to 'chain' upon it..." #
  • Most jQuery methods return the jQuery Object itself, which contains the elements you last selected.
  • This means that you can call methods using dot notation one after another with no new statements.
$("#chainability") // The id of the current slide.
    .fadeOut("slow")
    .fadeIn("fast")
    .slideUp(500)
    .slideDown(1000)
    .toggleClass("highlight")
    .click(function() { alert("Greetings from the chain!"); });

JSONP—JSON with Padding!

  • JSON is a data-interchange format that is actually valid JavaScript.
  • In order to get around Ajax domain limitations, somebody came up with a great idea.
  • JSONP (basically) generates a new <script /> tag, and loads a remote src url.
  • Based on the URL, the server then generates JSON and wraps it in a JavaScript function call.
  • When the script is loaded, it executes the function and we have access to the data.
  • Let's look at how we can interact with Flickr in this way.

Everything's an Object

JavaScript is an Object-Oriented Language.

Most values in JavaScript are objects, which means they can have properties and methods.

console.log("What did she say?".length);
console.log(["A", "B", "C"].join(" -> "));

Arbitrary properties and methods can be created:

var foo = {};
foo.awesome = true;
console.log(foo.awesome);

var bar = ['All', 'Sorts', 'of', 'Values'];
bar.howLongIsThisArray = function() {
    return this.length;
};
alert(bar.howLongIsThisArray());

Primitives vs. Constructors

Primitive types, in JavaScript, are built-in, basic types, which have special chracter sequences to define them.

// Some Primitives
var arr = [],
    obj = {},
    str = "",
    bool = true;

Most primitives also have Constructors, which are a formal way of creating objects. Concstructors make use of the new operator.

// Some Constructors
var arr = new Array(),
    obj = new Object(),
    str = new String("""),
    bool = new Boolean(true);

Personally, I rarely use constructors, as they are typically unnecessary bloat.

this

  • When a function run, this is set to the global (window) object.
  • The window object is called the global object because it contains all the global functions and variables.
    (e.g., window.console.log() === console.log())
function foo() {
    alert(this);
}
foo(); // alerts reference to window

this in a Method

  • Heading deeper down the rabbit hole, we come to the method.
  • When a function is a method of an object and called, this behaves differently.
  • Instead of this being a reference to the window object, it's a reference to the object of which it's a method.
var user = {
	name: "Johnny Alabaster",
	age: 43
};
user.getInfo = function() {
    alert(this);
    alert(this.name);
    alert(this.age);
};
user.getInfo(); // alerts reference to user

this as a Constructor

  • If your mind has yet to be blown, this will do it.
  • When a function is called with the new operator, a new object with the properties and methods of the function is created and assigned to this.
  • If a specific value is not returned from the function, then this will be returned.
function User(name, password) {
	this.name = name;
	this.password = password;

	this.getInfo = function() {
		alert(this.name);
        alert(this.password);
	};
}
var user1 = new User("peterwooley", "Freedom");
var user2 = new User("thor", "HammerTime");

// logs both users names and passwords
user1.getInfo();
user2.getInfo();
